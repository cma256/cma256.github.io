Inefficient Workflow Processor Cache Look Ups

A duplicate cache lookup and repeated cache look ups sharing a common prefix in the "workflow processor" were emitting billions of unnecessary, daily cache lookups. The duplicated cache lookups could be cached in-memory and reused. The repeated cache look ups sharing a common prefix could be rewritten to use a bulk cache lookup operation. Pull requests #115863 and #115864 were opened to remediate these inefficiencies.

Discovery

The performance issue was discovered using Sentry's tracing tool. Using the "aggregates" table and sorting by high span-emitting traces, I was able to triage subroutines by volume. Clicking into individual traces showed duplicate lookups and repeated lookups for cache keys with a common prefix. Inspecting the code location revealed the defects.

Results

After merging the pull request, the workflow processor subroutine stopped emitting 60,000 cache look ups per second.
