Remove Unnecessary Redis Script Existence Check

The span buffer is an extremely high-throughput span buffering process. It has complex interactions with Redis which are modeled in a Lua script. The Lua script needs to be loaded into Redis prior to inserting spans. The Lua script's existence in Redis was being checked on every "span group" insertion. However, its easier to ask for forgiveness than permission. We can load the script once, trust its there, and reload the script only in the event a "NoScriptError" was raised. Pull request #117427 was opened to address this inefficiency.

Discovery

The span buffer is a known, high-throughput process with tight performance requirements. Analysis was performed based on my intuition and prior experience. The analysis process required a careful reading of the code and documentation of the call graph.

Results

After merging the pull request, span group insertion time declined by 83% (from 150ms to 26ms on average). Total processing time per span declined by 50%.
