Typed, Partial Deserialization with Msgspec

The span buffer is an extremely high-throughput span buffering process. It receives JSON with a known schema enforced by the upstream producer. The span buffer was using the "orjson" library to deserialize the JSON payload, however, not every key in the payload was being used. Of the keys that were used, post-deserialization validation steps were applying type narrowing assertions. Pull request #117673 added typed, partial deserialzation with msgspec. This enabled the span buffer to only parse the ~500 bytes it needed and ignore the remaining span data (up to 10 megabytes).

Discovery

The span buffer is a known, high-throughput process with tight performance requirements. Analysis was performed based on my intuition and prior experience.

Results

After merging the pull request, parsing time declined by 300% (from 18us to 6us on average).
